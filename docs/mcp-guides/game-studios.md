# AI Game Class Studios 使用指南

## 简介

AI Game Class Studios 是一个游戏开发 AI 代理框架，包含 48 个专业 AI 代理，覆盖游戏开发全流程。

## 核心功能

### 1. 引导式工作流

输入 `/start` 开始引导式入门，AI 会询问你的项目状态并推荐合适的工作流程。

### 2. 可用命令一览

#### 项目启动阶段

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/start` | 引导式入门 | 首次使用，不知道从哪开始 |
| `/onboard` | 项目初始化 | 创建新项目结构 |
| `/setup-engine` | 引擎配置 | 配置项目使用的游戏引擎 |

#### 设计阶段

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/brainstorm` | 创意头脑风暴 | 生成游戏创意、机制想法 |
| `/quick-design` | 快速设计 | 生成设计文档草稿 |
| `/create-architecture` | 架构设计 | 设计技术架构 |
| `/ux-design` | UX 设计 | 设计用户界面和体验 |
| `/art-bible` | 美术风格指南 | 创建美术风格文档 |

#### 开发阶段

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/create-epics` | 创建史诗任务 | 分解大型功能 |
| `/create-stories` | 创建用户故事 | 细化开发任务 |
| `/dev-story` | 开发故事 | 按故事进行开发 |
| `/prototype` | 原型开发 | 快速验证概念 |

#### 测试阶段

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/qa-plan` | 测试计划 | 制定测试策略 |
| `/smoke-check` | 冒烟测试 | 快速验证核心功能 |
| `/bug-report` | Bug 报告 | 记录和追踪问题 |
| `/bug-triage` | Bug 分级 | 评估 Bug 优先级 |

#### 发布阶段

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/release-checklist` | 发布清单 | 检查发布前准备 |
| `/launch-checklist` | 上线清单 | 上线前检查 |
| `/changelog` | 更新日志 | 生成版本更新说明 |
| `/patch-notes` | 补丁说明 | 生成补丁说明 |

#### 代码质量

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/code-review` | 代码审查 | 审查代码质量 |
| `/design-review` | 设计审查 | 审查设计方案 |
| `/tech-debt` | 技术债务 | 记录和处理技术债务 |
| `/security-audit` | 安全审计 | 检查安全问题 |

#### 团队协作

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/sprint-plan` | 冲刺计划 | 规划开发周期 |
| `/sprint-status` | 冲刺状态 | 查看当前进度 |
| `/retrospective` | 复盘会议 | 总结经验教训 |
| `/estimate` | 工时估算 | 评估任务时间 |

## 使用示例

### 创建新游戏项目

```
你: /start

AI: 欢迎来到 Claude Code Game Studios！请问你现在处于什么阶段？
    A) 还没有想法
    B) 有模糊的想法
    C) 有清晰的概念
    D) 已有现有工作

你: B

AI: 好的，你有一些模糊的想法。让我们用头脑风暴来细化...
    [运行 /brainstorm]
```

### 开发功能

```
你: 我想要实现一个背包系统

AI: 好的，让我帮你设计背包系统。我会：
    1. 创建功能设计文档
    2. 分解为开发任务
    3. 生成代码框架
    
    可以开始吗？

你: 可以

AI: [创建设计文档和代码...]
```

## 专业团队代理

Game Studios 包含多个专业团队代理：

- **team-level** - 关卡设计团队
- **team-combat** - 战斗系统团队
- **team-narrative** - 叙事设计团队
- **team-ui** - UI/UX 团队
- **team-audio** - 音频团队
- **team-qa** - 质量保证团队
- **team-release** - 发布团队
- **team-live-ops** - 运营团队

调用示例：

```
你: /team-level 帮我设计一个新手关卡

AI: [关卡设计团队代理启动...]
```

## 项目结构

Game Studios 会在项目中创建以下结构：

```
项目根目录/
├── CLAUDE.md           # 项目主指令文件
├── .claude/            # Claude Code 配置
│   ├── agents/         # AI 代理定义
│   ├── skills/         # 可用命令
│   ├── rules/          # 开发规则
│   └── settings.json   # 设置文件
├── design/             # 设计文档
│   ├── gdd/           # 游戏设计文档
│   └── art/           # 美术设计
├── production/         # 生产管理
│   ├── sprints/       # 冲刺记录
│   └── milestones/    # 里程碑
├── prototypes/         # 原型代码
└── src/               # 源代码
```

## 常见问题

### Q: 命令不生效？

确保在正确的项目目录下启动 Claude Code，并且目录中存在 `.claude` 文件夹。

### Q: 如何自定义代理？

编辑 `.claude/agents/` 目录下的代理定义文件。

### Q: 如何添加新命令？

在 `.claude/skills/` 目录下创建新的 `命令名/SKILL.md` 文件。
